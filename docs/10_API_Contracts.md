このドキュメントは、GraphQL（公開契約）と gRPC/Proto（内部契約）の管理・バージョニング・互換性ルールを定義します。

## 1. ソース・オブ・トゥルース

- GraphQL: `backend/schema.graphql`
- gRPC: `proto/**/v1/*.proto`

## 2. 互換性ポリシー

### 2.1 GraphQL（後方互換維持）

- 追加: フィールド/型/引数の追加は後方互換（Nullable を基本）
- 変更: 型変更/必須化は破壊的変更のため禁止。新フィールドで代替
- 削除: 非推奨（@deprecated）→クライアント移行後に削除

### 2.2 gRPC/Proto（後方互換維持）

- 追加: 新フィールドは新しい番号で追加（既存番号は不変）
- 変更: フィールド型/番号の変更は不可。新メッセージ/メソッドで代替
- 削除: `reserved` で番号/名前を確保の上で削除

## 3. バージョニング

- GraphQL: スキーマファイルは単一。互換性を破る場合はサーバーに複数フィールドを併存
- Proto: 破壊的変更は `v2` パッケージとして追加し、移行期間中は v1 と並行稼働

## 4. コード生成

- GraphQL: Codegen（TypeScript 型/operations）を Web/RN で共有
- gRPC: Go のサーバー/クライアントコードを CI で生成

## 5. 契約テスト

- GraphQL: スナップショット/型整合テスト、主要クエリ/ミューテーションのスモーク
- gRPC: サーバー/クライアントの生成が最新であること、代表メソッドの結合テスト

## 6. 認証/メタデータ

- GraphQL: Cognito JWT を BFF で検証（`@auth`）
- gRPC: `x-user-id`, `x-email`, `authorization` の伝播を規定
