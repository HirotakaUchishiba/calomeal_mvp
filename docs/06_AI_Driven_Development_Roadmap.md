このドキュメントは、あなた（ユーザー）がプロジェクトマネージャー兼リードエンジニアとして、AI（熟練プログラマー）を指揮し、本MVPを完成させるための、より具体的で実践的なロードマップです。

あなたの役割はコードを書くことではなく、設計資料に基づいた的確な指示（プロンプト）をAIに与え、生成されたコードをレビューし、プロジェクト全体を統合していくことにあります。

## フェーズ0: 基盤構築 - AIとの共同作業環境を整える

**目的**: コーディングを開始するための全ての準備をAIに指示し、自動化された開発基盤を構築します。

### 1. ローカル開発環境の構築

* **あなたのタスク**:
    1.  AIに`docker-compose.yml`の作成を指示します。
    2.  生成されたファイルを確認し、ローカルで`docker-compose up`コマンドを実行して環境が立ち上がることを確認します。

* **AIへの指示プロンプト例**:
    ```
    『実装・実行ブループリント』の技術スタックに基づき、以下の要件を満たすdocker-compose.ymlファイルを作成してください。
    1. サービス: backend (Go), db (PostgreSQL), frontend (React/Node.js)
    2. PostgreSQLの接続情報は、ルートディレクトリの.envファイルから読み込むように設定してください。
    3. GoとReactのソースコードは、ローカルマシンからコンテナにマウントし、ホットリロードが可能な構成にしてください。
    ```

### 2. クラウドインフラの骨格構築 (IaC)

* **あなたのタスク**:
    1.  AIにTerraformのプロジェクト構造を生成させます。
    2.  生成されたファイル群をリポジトリに追加します。

* **AIへの指示プロンプト例**:
    ```
    『インフラ・DevOps・運用設計資料』の「1.1. Terraform プロジェクト構造」で示されているモジュールベースの構成に従い、Terraformプロジェクトのディレクトリと空の.tfファイルを生成してください。
    ```

### 3. CI/CDパイプラインの初期設定

* **あなたのタスク**:
    1.  AIにGitHub Actionsの基本ワークフローを作成させます。
    2.  生成されたYAMLファイルを`.github/workflows/`ディレクトリに配置します。

* **AIへの指示プロンプト例**:
    ```
    『インフラ・DevOps・運用設計資料』に基づき、developブランチへのプルリクエストがマージされた際に、フロントエンドのLintとバックエンドの単体テストを実行するGitHub Actionsのワークフローファイルを作成してください。
    ```

## フェーズ1: ユーザー認証の実装 - APIからUIへ

**目的**: アプリケーションの入口となるユーザー登録・ログイン機能を、バックエンドからフロントエンドへと順を追って実装させます。

### 1. データベースとモデルの定義

* **あなたのタスク**:
    1.  AIにDBマイグレーションファイルの作成を指示します。
    2.  そのマイグレーションに対応するGoのデータモデル（構造体）を生成させます。

* **AIへの指示プロンプト例**:
    ```
    1. 『実装・実行ブループリント』の「4.4. データベーススキーマ」に基づき、users, user_profiles, user_goalsテーブルを作成するためのSQLマイグレーションファイルを生成してください。
    2. 次に、今生成した3つのテーブルに対応するGoの構造体を定義してください。
    ```

### 2. バックエンドAPIの実装

* **あなたのタスク**:
    1.  GraphQLスキーマ、サービス層、リゾルバ層の順に、一つずつコード生成を指示します。
    2.  生成されたコードを統合し、単体テストでエラーが出ないことを確認します。

* **AIへの指示プロンプト例**:
    ```
    1. 『実装・実行ブループリント』のGraphQLスキーマを参考に、ユーザーオンボーディング用のcompleteOnboardingミューテーションを含むschema.graphqlsを作成してください。
    2. 次に、Cognitoと連携してユーザー情報をDBに保存するUserServiceのインターフェースとスタブ実装をGoで作成してください。
    3. 最後に、completeOnboardingミューテーションのリクエストをUserServiceに渡すGraphQLリゾルバを実装してください。
    ```

### 3. フロントエンド画面とロジックの実装

* **あなたのタスク**:
    1.  ワイヤーフレームに基づき、UIコンポーネントを一つずつ生成させます。
    2.  生成したコンポーんねんとAPI連携ロジックを追加させます。
    3.  ローカルで動作確認します。

* **AIへの指示プロンプト例**:
    ```
    1. 『プロダクト・UI/UX設計資料』のワイヤーフレームに基づき、SignUpPage.tsx, LoginPage.tsx, OnboardingPage.tsxの3つのReactコンポーネントを生成してください。スタイリングは不要です。
    2. OnboardingPage.tsxに、フォームの入力値をcompleteOnboardingミューテーションとしてバックエンドに送信するロジックを、Apollo Clientを用いて実装してください。
    ```

## フェーズ4: 自動化と品質保証

**目的**: AIにテストコードとデプロイパイプラインを構築させ、手動作業をなくし、アプリケーションの品質と信頼性を高めます。

### 1. テストコードの生成

* **あなたのタスク**:
    * 主要なビジネスロジックに対する単体テストコードを生成させます。
    * 主要なユーザーストーリーをカバーするE2Eテストコードを生成させます。

* **AIへの指示プロンプト例**:
    ```
    1. Goで実装したLogServiceの栄養計算ロジックに対する単体テストを作成してください。正常系と異常系の両方をカバーしてください。
    2. 『インフラ・DevOps・運用設計資料』の「第5部: E2Eテストケース仕様」で定義されたシナリオ (E2E-HP-001)を、Playwrightを使って実装してください。
    ```

### 2. CI/CDパイプラインの完成

* **あなたのタスク**:
    1.  フェーズ0で作成したCI/CDパイプラインに、ビルド、デプロイのステップを追加させます。
    2.  `develop`ブランチへのマージで自動的にデプロイされることを確認します。

* **AIへの指示プロンプト例**:
    ```
    フェーズ0で作成したGitHub Actions ワークフローに、以下のステップを追加してください。
    - バックエンドのDockerイメージをビルドし、AWS ECRにプッシュする。
    - データベースマイグレーションを実行する。
    - Terraformを適用し、新しいコンテナイメージをAWS Fargateにデプロイする。
    ```
